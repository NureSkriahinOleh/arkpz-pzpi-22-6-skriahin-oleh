1.2  Хід роботи
1.2.1  Було проаналізовано бізнес-вимоги до системи пожежної безпеки для торгових центрів, визначено основні ризики, потенційних клієнтів та їхні потреби. Досліджено актуальність теми, враховуючи зростання екологічних і техногенних ризиків.
1.2.2  Проведено аналіз існуючих аналогів на ринку IoT-рішень у сфері безпеки. Визначено переваги запропонованої системи, такі як комплексний підхід до моніторингу, доступна вартість компонентів та адаптивність.

Під час виконання роботи було описано концепцію IoT-системи для пожежної безпеки торгових центрів, сформовано рамки проєкту, проведено аналіз ринку та визначено конкурентні переваги продукту.

1.	Business Requirements
1.1	Background 
Сучасні торгові центри є об’єктами підвищеного ризику через велику кількість відвідувачів, наявність складних інженерних систем та ризик виникнення надзвичайних ситуацій, таких як пожежі, витоки газу або задимлення. Традиційні системи пожежної безпеки часто реагують із затримкою, не забезпечуючи оперативного контролю над ситуацією.
Мета проекту: розробити IoT-систему, яка забезпечить у реальному часі моніторинг подій у торгових центрах за допомогою камер відеоспостереження, датчиків руху, диму, температури, газу та вологості. Це дозволить знижувати ризики шляхом своєчасного виявлення загроз та забезпечення ефективного реагування.
Цей продукт допоможе підвищити рівень безпеки, зменшити матеріальні втрати та зберегти людські життя в умовах надзвичайних ситуацій.
1.2  Business Opportunities
1.2.1  Аналіз конкурентів
Існують кілька значущих конкурентів на ринку IoT-систем для пожежної безпеки, зокрема:
•	Honeywell Connected Safety – спеціалізується на інноваційних рішеннях у сфері безпеки, пропонуючи високоякісні датчики та інтеграцію із системами управління будівлями. Однак їхні продукти є дорогими, що робить їх недоступними для середніх та невеликих торгових центрів.
•	Bosch Building Technologies – пропонує комплексні рішення з моніторингу безпеки, включаючи відеоспостереження, проте їхні системи орієнтовані на великі об'єкти з високими бюджетами.
•	Siemens Smart Infrastructure – інтегрує датчики та системи управління будівлями, але їхні продукти вимагають значних витрат на впровадження та обслуговування, що ускладнює масштабування.
•	Ajax Systems – пропонує конкурентоспроможні бездротові системи для домашньої та комерційної безпеки. Проте їхні продукти більше підходять для локальних рішень і не орієнтовані на великі об’єкти з комплексними потребами.
1.2.2  Переваги розроблюваної системи
1.	Інтегрований підхід: Об’єднання відеоспостереження та різноманітних датчиків (диму, температури, вологості, газу) для створення повної картини подій.
2.	Висока адаптивність: Можливість додавання нових датчиків або інтеграції із вже наявними системами у торговому центрі.
3.	Доступна вартість: Завдяки використанню доступних компонентів і хмарних технологій розроблювана система буде значно дешевшою у впровадженні та обслуговуванні, ніж рішення від Honeywell чи Siemens.
4.	Простота впровадження: Система потребує мінімальних змін у наявній інфраструктурі.
1.3  Модель монетизації
1.	Продаж ліцензій на систему для торгових центрів.
2.	Щомісячна абонплата за хмарний сервер для зберігання даних і аналітики.
3.	Надання додаткових платних послуг, таких як навчання персоналу або інтеграція з існуючим обладнанням.

2.	Vision of the Solution
2.1 Vision Statement
Розроблювана система забезпечує комплексний підхід до управління пожежною безпекою в торгових центрах, використовуючи сучасні технології IoT. Продукт об'єднує камери відеоспостереження, датчики руху, диму, температури, газу та вологості для оперативного моніторингу та реагування на небезпечні ситуації.
Система дозволяє:
•	Забезпечити безперервний моніторинг подій у реальному часі.
•	Швидко виявляти потенційні загрози, такі як пожежі чи витік газу.
•	Надсилати миттєві сповіщення до мобільного додатку для інформування охоронців, співробітників та відвідувачів.
•	Автоматизувати процеси збору, зберігання та аналізу даних для подальшого підвищення ефективності безпекових заходів.
Реалізація цієї системи сприятиме мінімізації ризиків у торгових центрах, підвищенню рівня безпеки та створенню комфортного середовища для відвідувачів і персоналу.

2.2  Major Features
•	MF-1: Підключення камер відеоспостереження та різних типів датчиків (руху, диму, температури, газу, вологості) для безперервного моніторингу у реальному часі.
•	MF-2: Автоматичне надсилання сповіщень до мобільного додатку у разі виявлення небезпечної ситуації (пожежа, витік газу тощо).
•	MF-3: Збереження відеозаписів та даних від датчиків у базі даних для подальшого аналізу.
•	MF-4: Функція тривожної кнопки у мобільному додатку для негайного виклику охорони у разі виявлення загрози.
•	MF-5: Автоматична генерація звітів про безпеку, включно з деталями про інциденти, статистикою активності датчиків та інформацією про порушення.
•	MF-6: Візуалізація даних у зручному інтерфейсі для керівництва та охоронного персоналу, включно зі звітами, історією подій та доступом до записів із камер.
Ці функції створюють цілісну систему для підвищення пожежної безпеки, забезпечуючи як оперативну реакцію, так і стратегічний аналіз для покращення безпекових процесів.

3.  Scope and Limitations
3.1  Scope of Initial Release
3.1.1  Серверна частина
Серверна частина виконує ключову роль у системі управління пожежною безпекою торгових центрів. У рамках першого випуску буде реалізовано такі функції:
•	Моніторинг стану безпеки: отримання даних від IoT-пристроїв, таких як димові датчики, теплові сенсори та датчики газу.
•	Обробка аварійних ситуацій: аналіз отриманих даних для виявлення потенційних загроз і автоматична генерація сигналів тривоги.
•	Інтеграція з аварійними службами: передача критичних даних до відповідних служб (пожежна охорона, служба безпеки) у разі надзвичайної ситуації.
•	Система сповіщень: інформування адміністрації торгового центру через email або push-сповіщення про виявлені ризики чи порушення правил безпеки.
3.1.2  IoT-клієнт
IoT-клієнт забезпечуватиме взаємодію системи з датчиками та іншими пристроями. Основна функціональність IoT-клієнта включає:
•	Збір даних із сенсорів: обробка показників датчиків диму, температури, вологи, рівня газу та інших параметрів.
•	Передача даних на сервер: синхронізація показників у реальному часі для оперативного реагування на зміну умов.
•	Оновлення пристроїв: можливість дистанційного оновлення програмного забезпечення сенсорів через сервер.
•	Тестування системи: перевірка працездатності датчиків та їхнього з’єднання з сервером для гарантії безперебійної роботи системи.

3.2  Scope of Subsequent Releases
3.2.1  Веб-клієнт
Веб-клієнт надасть адміністраторам доступ до управління системою через браузер. Основні функції веб-клієнта включають:
•	Особистий кабінет адміністратора: можливість переглядати стан пристроїв, отримувати сповіщення про аварії та керувати налаштуваннями системи.
•	Моніторинг безпеки: інтерактивна карта, яка показує стан датчиків у реальному часі.
•	Управління пристроями: активація або деактивація сенсорів, перевірка їхнього статусу та оновлення програмного забезпечення.
•	Звіти та аналітика: формування детальних звітів про інциденти, стан безпеки та ефективність роботи пристроїв.
•	Багатокористувацька підтримка: розподіл ролей для різних користувачів (адміністраторів, технічного персоналу, менеджерів).
3.2.2  Мобільний клієнт
Мобільний клієнт буде розроблений для платформ Android дозволить користувачам оперативно отримувати доступ до системи. Основна функціональність мобільного клієнта включає:
•	Сповіщення про аварії: push-сповіщення про критичні події, які потребують негайного реагування.
•	Дистанційний моніторинг: перегляд стану сенсорів у реальному часі та отримання інформації про небезпечні ділянки.
•	Інтерактивна карта: швидкий доступ до розташування датчиків у приміщенні торгового центру.
•	Реагування на інциденти: можливість миттєво викликати аварійні служби або активувати локальні засоби пожежогасіння через мобільний інтерфейс.
•	Користувацький інтерфейс: інтуїтивно зрозуміле меню для швидкого доступу до основних функцій.
Такі функції у веб- і мобільному клієнтах дозволять забезпечити повний контроль над системою та оперативне реагування на небезпечні ситуації.

4.  Business Context
4.1  Operating Enviroment
Розроблювана система використовує сучасні технології та інструменти для забезпечення її функціонування. Для створення серверної частини застосовується Django REST Framework, що забезпечує створення надійного API. Дані зберігаються в базі даних PostgreSQL, а інтерфейс користувача реалізовано за допомогою React.js. У процесі розробки використовуються такі інструменти, як Visual Studio Code або інший текстовий редактор, система управління версіями Git, а також, за потреби, Docker для спрощення розгортання.
Для забезпечення роботи системи застосовуються різні бібліотеки. У серверній частині це JWT-Token для авторизації користувачів і djangorestframework, а у фронтенді – Axios для обміну даними з API та Bootstrap для стилізації інтерфейсу.
Система має відповідати ряду вимог. Вона повинна бути доступною для користувачів у будь-який час, працювати безперервно 24/7. Надійність забезпечується регулярним резервним копіюванням бази даних і моніторингом серверів для мінімізації простоїв. Для досягнення високої продуктивності передбачено швидкий час відповіді сервера і оптимізацію бази даних, наприклад, за рахунок використання індексів. Особлива увага приділяється цілісності даних: паролі зберігаються в зашифрованому вигляді, а чутлива інформація, якщо необхідно, шифрується. Дані зберігаються централізовано на сервері.
